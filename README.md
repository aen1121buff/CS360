# CS360
Requirements and goals of the app, and the user need it addressed
This mobile app was designed to help users manage simple, everyday information in one place with a clear, low friction workflow. The core goal was to provide a functional, reliable experience for creating, viewing, and maintaining user data without unnecessary complexity. The app focuses on meeting the user’s need for an organized, easy to navigate tool that makes common actions fast to complete, while also maintaining a consistent look and predictable behavior across screens.

Screens and features needed for a user centered UI, and why the design worked
To support user needs, the app required a login or entry point, a primary home or dashboard screen, and task focused screens where users could add, edit, view, and remove information. It also needed supporting UI elements like clear navigation, readable layouts, consistent button placement, and meaningful feedback (for example confirmations, error messages, and empty states). The UI designs kept users in mind by prioritizing clarity over density, using consistent labels and spacing, and ensuring the most common actions were the easiest to find and complete. The designs were successful because they reduced user effort, prevented confusion through consistency, and made the app feel predictable, which is a key part of usability.

Coding approach, techniques used, and how they apply in the future
I approached development by starting from the approved UI baseline and building the app in small, testable increments. I broke requirements into features, implemented one feature at a time, and regularly verified behavior before moving forward. Key strategies included keeping code modular, separating UI concerns from logic where possible, using consistent naming and structure, and following platform best practices for layouts, navigation, and data handling. These techniques apply directly to future projects because they reduce risk, make debugging easier, and keep the codebase maintainable as features grow.

Testing process, why it matters, and what it revealed
I tested functionality by validating each feature against the requirements and running through realistic user flows, including normal cases and edge cases. This included checking navigation between screens, verifying that inputs were handled correctly, confirming that updates persisted as expected, and ensuring the UI behaved correctly under different conditions (empty lists, missing inputs, invalid values). Testing is important because it prevents defects from reaching users, confirms the app meets its intended purpose, and reveals issues that are easy to miss during initial implementation, such as unexpected states, validation gaps, or UI logic inconsistencies.

Innovation and challenge overcome during the full process
Across planning, design, and implementation, the main challenges came from turning a static UI plan into a working app while keeping the experience consistent and user centered. I had to innovate by adjusting implementation details to fit real platform constraints while still matching the intended design, and by resolving integration issues where components interacted (navigation, data updates, and user feedback). This required problem solving, iteration, and making tradeoffs that preserved usability and app stability.

Component where I demonstrated the most success
I was particularly successful in implementing the end to end flow that connects the user interface to functional behavior, where users can complete the main actions the app was designed for. This includes building screens that match the UI design, wiring them together with consistent navigation, and ensuring the app responds correctly to user input and state changes. This part of the project best demonstrates my ability to apply mobile development best practices, translate a user centered design into working software, and deliver a cohesive experience.
